목적: 설계서 v1.0의 플랫폼, 기초, 전표, 보조부, SA 기준 182개 기능 화면, 세무/결산/자금/고급회계, 운영·보안·API·배치·부록 내용을 모두 수용할 수 있는 메뉴 구조를 정의한다.

1. 표기 범례

BF-09 기초/기준정보 화면 JV-01 전표 입력 화면 AUX-02 보조부/보조원장 SA-VAT SA 기능 화면군 Step-up 재인증 필요 관리자 관리자/권한자 전용 기장 기장 모드 영향 선택 기능 플래그·업종·Phase별 노출 회색 들여쓰기: 화면 내 탭/하위 기능 또는 설계 추적 항목
메뉴는 제품 화면 기준으로 재구성했다. 설계서의 기술 장(데이터 모델, API, 배치, 검증, 보안, 운영)과 부록은 사용자 메뉴가 아니라 운영/관리/설계 추적 메뉴로 연결한다.

2. 4단 메뉴 구조도

1뎁스 채널 → 2뎁스 상위 메뉴 → 3뎁스 하위 메뉴 → 4뎁스 세부 기능/화면
각 메뉴는 반드시 4단으로 해석한다. 1뎁스는 채널(운영콘솔/업무화면/공통), 2뎁스는 업무 영역, 3뎁스는 사용자가 진입하는 메뉴, 4뎁스는 화면·탭·실행 기능이다.

6. 메뉴 운영 규칙

  • 미노출 원칙: 권한 없는 메뉴는 비활성이 아니라 미노출한다. 요금제/기능 플래그 미포함 기능은 정책에 따라 업그레이드 안내로 노출할 수 있다.
  • 기장 모드: `TENANT_LED`는 이용회사 입력 중심, `OPERATOR_LED`는 운영자 기장 워크벤치와 회사 확인 화면 중심, `HYBRID`는 분담 규칙에 따라 입력/검토 권한을 나눈다.
  • 마감 상태: 메뉴는 유지하되 마감 기간 데이터의 신규/수정/삭제/기표 행위는 `CLOSING_LOCKED`로 차단한다.
  • 상태 불변성: `POSTED`, `CLOSED`, `LOCKED` 전표는 직접 수정/삭제하지 않고 역분개, 조정전표, 복구/이관 전용 메뉴로 처리한다.
  • 기능 플래그: 외화, 관계사, Cost-plus, 예산, 그룹보고, 제조원가/재고, XBRL, 다중 회계기준은 구독/업종/기능 플래그와 연동한다.
  • 감사 추적: 권한, 메뉴, 기장 모드, 접근 세션, 전표, 증빙, 데이터 이관, 신고 전송은 append-only 이력과 사유를 남긴다.
