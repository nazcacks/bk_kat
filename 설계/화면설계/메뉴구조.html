작성일: 2026-06-30 · 파일명: 설계/화면설계/메뉴구조.html

운영콘솔과 업무화면을 분리하고, 각 메뉴를 구분 → 상위 메뉴 → 하위 메뉴 → 세부 메뉴/화면의 4단 구조로 상세화했다. 업무화면에는 SA 기준 182개 세부 기능 화면을 개별 행으로 포함한다.

1. 구성 기준

운영콘솔 그리드 업무화면 그리드 공통 운영 규칙
0운영콘솔 메뉴 행
0업무화면 메뉴 행
182SA 세부 기능 화면
4메뉴 단계: 구분 / 상위 / 하위 / 세부
컬럼은 메뉴코드, 1단계 구분, 2단계 상위 메뉴, 3단계 하위 메뉴, 4단계 세부 메뉴/화면, 화면 ID, 업무 목적, 주요 데이터/엔티티, 권한/통제, 노출조건, 설계 근거 순서다. 같은 구분·상위·하위 메뉴는 행 병합으로 묶어 4단 구조가 바로 보이게 구성했다.

2. 운영콘솔 4단 메뉴 그리드

No 메뉴코드 1단계 구분 2단계 상위 메뉴 3단계 하위 메뉴 4단계 세부 메뉴/화면 화면 ID 업무 목적 주요 데이터/엔티티 권한/통제 노출조건/비고 근거

3. 업무화면 4단 메뉴 그리드

No 메뉴코드 1단계 구분 2단계 상위 메뉴 3단계 하위 메뉴 4단계 세부 메뉴/화면 화면 ID 업무 목적 주요 데이터/엔티티 권한/통제 노출조건/비고 근거

4. 공통 운영 규칙

규칙적용 메뉴처리 기준
권한 없는 메뉴 미노출전체메뉴 노출은 `RoleMenuPermission`, 버튼/행위는 `MenuActionGrant`, 데이터 범위는 Role의 `dataScope`와 가드레일로 평가한다.
기장 모드 연동OP-08, TN 전표/마감/신고/보고`TENANT_LED`, `OPERATOR_LED`, `HYBRID`에 따라 입력·검토·승인·확인(Ack) 행위를 분리한다.
Step-up 인증권한 변경, 신고 전송, 마감해제, 긴급대행, 대량 다운로드재인증 결과와 사유를 세션 및 감사 로그에 기록한다.
마감 잠금전표, 보조부, 신고, 결산마감 기간 신규/수정/삭제는 `CLOSING_LOCKED`로 차단한다. 정정은 역분개 또는 조정전표로 처리한다.
기표 후 불변성전표·원장·보조부`POSTED`, `CLOSED`, `LOCKED` 전표는 직접 수정/삭제 금지. 역분개, 삭제전표복구, 데이터관리 메뉴로만 처리한다.
기능 플래그외화, 관계사, 예산, 그룹보고, 제조원가/재고, XBRL구독, 업종, Phase, 회사 설정에 따라 메뉴를 노출하거나 업그레이드 안내로 표시한다.
감사 추적전체 중요 메뉴전표, 신고, 권한, 접근, 기장모드, 메뉴버전, 데이터 이관은 append-only 이력과 변경 사유를 남긴다.
