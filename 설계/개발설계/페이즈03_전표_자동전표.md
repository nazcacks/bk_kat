## 1. 페이즈 개요

### 1.1 목표

- 회계 원장의 원천이 되는 **수기 전표 입력 2종**(일반전표 SA-JNL-01, 매입매출전표 SA-JNL-02)과, 국세청 홈택스·여신금융협회·카드사·은행에서 증빙을 **자동수집→분개 추천→검증→전표전송**하는 **자동전표 파이프라인 9종**(SA-ATX-01~09)을 구축한다(§25.4, §25.5).
- 주간정기회의(2026-07-02) 결정에 따라 **일반전표입력(WJV-01)이 MVP 최우선 화면**이며, 계정과목 관리(P01의 WJV-08/09)와 함께 최우선 개발한다(§0.7, §22, 00_개요 1절).
- 전표 도메인 표준: 더존식 일자별 채번(§11.1-(2)), entryCode 6종+확장 5종(§11.1-(5), §11.1.1-(13)), 4종 일자 구조(§11.1.1-(7)), 승인 워크플로우 옵션화(§11.1-(7)), 과세유형 단일진실원(§25.4.2), 자동전표 스테이징·상태머신(§25.5, §8.4).

## 4. 화면별 상세설계

### 4.1 SA-JNL-01 일반전표입력 (WJV-01) — 1차

- **목적**: 부가세 신고 관련 매입매출 거래 **이외의 모든 거래**를 입력하는 단일 진입 화면. 입금/출금 전표 메뉴를 별도로 두지 않고 통합 — 현금(101) 라인 위치로 입출금 자동 판별, 「구분」 컬럼에 대체/입금/출금 표시(§25.4.1, §11.1-(1)).
- **진입경로**: 전표/장부 › 일반전표입력. 진입 시 전기일(오늘)의 당일 전표 목록 자동 조회(일자별 마스터-디테일, §11.1-(1) S9). 「＋ 전표 추가」 또는 행 선택 후 하단 분개 입력, 「저장 후 계속 입력」 연속 기표 지원.
- **권한**: `tenant:journal:write`(자가) · `operator:journal:review`(대행 검토) (§13.3, 부록 통합본 WJV-01 메타).
- **API**: `GET /api/journals?fiscalYear&journalDate`(월별) / `?dateFrom&dateTo`(기간) · `POST /api/journals`(journalType=GENERAL) · `PUT /api/journals/{id}` · `POST /api/journals/{id}/submit`(결재요청·채번) (부록 통합본 WJV-01·3.1.1).
- **산출물**: 분개장·총계정원장·현금출납장·거래처원장·재무제표(시산표/BS/IS)·보조부(자금·어음·카드)·인쇄 전표(§25.4.1).

#### 4.1.7 우측 도킹 패널 — 관리항목 입력 (WJV-02, R15)

- 관리항목 입력은 팝업 대신 **우측 도킹 패널(한 화면)** 기본: 라인 선택 시 해당 계정의 관리항목(6 관리차원·자금·차량·카드·증빙)이 패널에 표시되고 그 자리에서 입력. 복잡 입력(어음 상세 등)만 보조 팝업 유지(§11.1.1-(15)).
- 탭 구성(부록 통합본 4장 WJV-02): **관리차원**(사업장*·코스트센터·부서·프로젝트·현장·사원 — `DimensionConfig`가 노출/필수 결정) / **자금**(받을어음·지급어음·차입금 — AUX 보조부, 상태머신 별도) / **차량**(업무용승용차 — 운행기록·세무 한도 연계) / **카드**(카드계좌 — 카드 보조부·매입원장·공제구분 연계) / **증빙**.
- **법인세조정 계정의 관리계정 설정**: 접대비·감가상각·업무용승용차 등 세무조정 대상 계정에 관리항목(차량·사원·증빙유형)을 연결 — 세무조정 계산(§11.5) 기초자료를 입력 시점에 확보(R15).
- 세금계산서 교부 후 카드결제 건은 세금계산서 기준으로 한 번만 매입세액 공제, 카드결제는 결제수단만(이중공제 방지, R16-④, 부록 통합본 4.3).
- 관리항목 마스터 등록·수정은 통합 관리 화면 WJV-07(P01 SA-BAS-07 영역)에서 수행 — 본 화면은 **선택**만(부록 통합본 4.5).

### 4.2 SA-JNL-02 매입매출전표입력 (WPV-01) — 1차

- **목적**: 부가세 신고 관련 매입매출 거래 입력 단일 화면. 한 전표가 **세무 신고자료(상단부)** 와 **재무회계 분개(하단부)** 를 동시 생성하는 **2계층 구조**(§25.4.2, 25.4.2.0).
- **식별·상태·API**: 복합키 (tenant_id, fiscal_year, journal_date, journal_no), 번호 **50001~99999**(일반전표 1~50000과 분리), 더존식 일자단위 채번·수동수정 불가. 상태 DRAFT→CLIENT_SUBMITTED→REVIEWING→PENDING_APPROVAL→APPROVED→POSTED→CLOSED. API `GET /api/journals?source=PURCHASE_SALE&fiscalYear&period&taxType` · `POST /api/journals`(journalType=PURCHASE_SALE) · `PUT /api/journals/{id}` · `POST /api/journals/{id}/auto-line`. 권한 `tenant:journal:write`·`operator:journal:review`(§25.4.2 화면 골격·식별).
- **업무흐름 5단계**: ① 기간·유형 조회 → ② 상단 세무헤더 입력 → ③ 부가세 자동라인 생성 → ④ 하단 분개라인 생성·보정 → ⑤ 검증·저장. 보조 흐름: 전자발행·더보기(복수거래 등)·홈택스 엑셀 업로드(§25.4.2 업무흐름).

상단 **과세유형(부가세 계정) 입력 시에만** 생성(BK 핵심개념 — 일반전표는 자동라인 비대상, §11.1-(3)). `vat_account_rule`+자동라인 생성기가 처리, 하단에 `자동` 표식·잠금.

| 구분 | 부가세 자동라인 | 채권/채무 라인 금액 |
|---|---|---|
| 매출 과세(11·14·17·22 등) | 대변 **부가세예수금** = 세액 | 외상매출금/현금/카드채권 = 공급대가 |
| 매입 과세(51·57·61 등) | 차변 **부가세대급금** = 세액 | 외상매입금/미지급금/카드채무 = 공급대가 |
| 영세·면세(12·13·52·53 등) | 생성 안 함 | 채권/채무 = 공급가액 |
| 불공(54) | 생성 안 함(불공제) | 비용·자산에 세액 포함(공급대가) |

#### 4.2.5 하단부(분개라인)·자동분개 엔진 (25.4.2.4)

| 필드 | 규칙 |
|---|---|
| 분개유형 | 0.분개없음/1.현금/2.외상/3.혼합/4.카드/5~7.외상추가1·2·3 — 자동분개 엔진 룰 입력값 |

#### 4.2.7 전자(세금)계산서 발행·전송 — 비고=발행정보 (25.4.2.6)

- 전자 값: 공란(발행 가능·미발행)/1.전자입력(수취 전자분 수기 입력)/전자발행. 과세기간 종료 다음달 11일 기준 전자/전자외 구분.
- `etax_status` 상태머신: 발행/전송 결과=전자입력·전자발행·국세청성공·전송실패·역발행 / 발행상태=확인요청·확인·발행대상·발행요청 / 신고상태=미전송·전송성공.
- [전자발행] 대상: 매출 11·12·13 / 매입 역발행 51·52·53·54. 기능: 화면구성·휴폐업조회·전자발행·발행취소(**당일 '확인요청' 전표만**)·이메일등록·발행옵션. 수량·단가 소수점 2자리(3자리 이상 반올림). 전송결과=외부 전자세금계산서 서비스·국세청 반영.
- **비고=발행 메타정보**: 신고 집계·분개 **미반영**(문서 인쇄 전용) — 상단 그리드가 아니라 **전자발행 모달 > 발행정보 > 비고**에서 입력. 비고·영수/청구·이메일·책(일련)번호·사업자단위과세는 양식 최상단 입력→본문 읽기 전용 연동. 승인번호는 미발급(발행 시 국세청 자동 채번). 복수거래 품목별 비고 열 지원. 수기 전자입력분 내부 메모는 하단 적요 사용. 저장: `etax_issue.remark`.

- 취합 2경로: ① 홈택스 직접 연동(ATX-02~05 전표전송→자동전표검증→확정), ② **홈택스 엑셀 업로드**(더보기 ㉒): 다운로드→자료구분·기간 선택→업로드·미리보기(표준 서식 상이 시 '서식 내려받기')→컬럼 매핑→검증→일괄 전표 생성(50001~ 채번).

- 검증·중복방지: 사업자번호 누락/형식오류 행 제외(차단), **중복 대조**(일자·사업자번호·공급가액·세액·승인번호 — 직접 연동 수집분·기입력 전표 대조, `DuplicateVoucherGuard`), 합계 정합, 유형 보정 확인. 구현 단위: `HometaxExcelImportCommand`·`ExcelColumnMappingProfile`·`TaxTypeInferenceRule`·`BulkVoucherGenerateCommand`.
