## 1. 페이즈 개요

### 1.1 목표

- Smart A Ch.9 "결산재무제표" 14메뉴 + Ch.10 "결산장부" 7메뉴(SA-CLS-01~21)를 **"결산/보고패키지 모듈"(결산 워크플로우 → 결산분개 자동생성 → 검증 → 보고패키지 산출)** 로 재설계한다(§25.9 서두, §25.10 서두).
- 결산대체분개는 `entryCode` 중 `CLOSING_DR`(결차)/`CLOSING_CR`(결대) 라인으로 생성하며, **결차/결대 라인만 제조원가명세서·손익계산서 매출원가·당기제품제조원가에 반영**한다(§25.9 서두, §11.1-(5) M2, §12.2).
- 마감을 단순 상태 전환이 아닌 **검증·승인 워크플로우**로 구현한다: 기간 상태 `FUTURE→OPEN→TEMP_CLOSED→CLOSED→LOCKED` + 마감해제 `REOPEN_REQUESTED→REOPENED`, 마감 체크리스트(`ClosingChecklist`) 게이트(§11.9, §12.2).
- 결산정리분개 자동화 8종(`ClosingAutoEntryRule`)과 **확정 전 미리보기·조정 전후 비교(M7)** 를 제공한다 — 미리보기는 G/L을 변경하지 않는다(§11.3).
- 확정 재무제표·시산표·명세서는 월별 보고패키지(`ReportPackage`, `PENDING→…→ACKED`)로 조립·검증·전달하며, 계정별 명세 tie-out(§11.14) 전건 통과 + 미결항목 0건이 검증 게이트다(§11.13).

### 1.3 선행조건·MVP 연관

- P03 전표(POSTED 승격·entryCode 6종)와 P04 장부 집계(원장·AgingSnapshot·은행 대사·AUX-06 ntsStatus 대사)가 결산 집계·마감 체크리스트의 원천이다(`00_개요` §2, §11.9-(2), §11.11).
- 마감·마감해제·결산 확정은 **주기장 측 권한**이다 — `TENANT_LED`는 이용회사 마감 권한자, `OPERATOR_LED`/`HYBRID`(주기장=OPERATOR)는 `BK_MANAGER`. 비주기장 측 시도는 `NOT_PRIMARY_BOOKKEEPER` 차단(§11.9-(3), §21).
- 회계기준 다중 지원(일반기업회계기준+K-IFRS+US GAAP+IFRS — 회사 선택), 병행원장 기준별 재무제표 각각 산출(§11.3).
- 제조원가·재고 수불 연동 원가계산(`ManufacturingCostRun`)은 **Phase 2**(기능 플래그 `manufacturingCostEnabled`) — 1차 제조원가명세서(05)는 전표·계정 집계 기반으로 구현(§11.21-(1), §25.9.5).

## 3. 업무 흐름

```
[P03/P04] POSTED 전표·원장·보조부 집계
   │
   ├─▶ ① 결산 전 검증: 마감 체크리스트(ClosingChecklist) — 미결/반려 전표·시산 차대·차원 필수값·
   │      스테이징 잔여·은행/카드 대사·미결항목(18.3)·tie-out(§11.14) (§11.9-(2))
   ├─▶ ② 결산 자동분개: ClosingAutoEntryRule 8종 실행 → 미리보기(조정 전후 비교 M7, G/L 미변경)
   │      → 확정 시 CLOSING_DR/CLOSING_CR 전표 생성 (§11.3)
   │      ├─ 경과·충당·리스: 기간배분 스케줄(11.15) 월별 안분/상각 전표
   │      ├─ 외화: 월말 외화평가(11.16) 미실현손익 + 익월 환산제거
   │      ├─ 대손: AgingSnapshot(P04) 기반 채권잔액×설정률
   │      └─ 비용 배부(N12): CostAllocationRule 실행 → 원가배부 전표(§11.19-(3))
   ├─▶ ③ 결산자료입력(CLS-01) [전표추가] → 시산표(02) 차대검증 → 재무상태표(03)/손익(04)/
   │      제조원가(05) → 이익잉여금처분(06, 손익대체) → 자본변동표(07) → 현금흐름표(08, 간접법+검증)
   ├─▶ ④ 마감: 가마감(TEMP_CLOSED) → 본마감(CLOSED, 주기장 결재+스냅샷) → 이용회사 Ack
   │      → 신고 완료 시 LOCKED. 마감해제 = Step-up+사유+승인(REOPEN) (§11.9-(3))
   ├─▶ ⑤ 보고패키지(11.13): 컴포넌트 생성 → 조립 → 검증(tie-out 전건+미결항목 0건) → 검토·결재
   │      → 전달 → Ack  [PENDING→GENERATED→VALIDATED→REVIEWED→DELIVERED→ACKED]
   │      └─ 부속명세서(09)·일괄출력(13)·경영정보(14)·주석 자동화(N10, §11.3.2)
   └─▶ ⑥ 세무 명세: 영수증수취(15)·송금명세(16)·지출증명합계표(17) → [P06~P08] 신고 연계
연차: 전 월차 본마감 + 결산분개 반영 → 연마감 → 차기 FiscalYear 생성·이월(§11.9-(3)5, P02/P10 연계)
```

### 4.1 [BK 공통] 마감 관리 워크플로우 — 회계기간·가마감/본마감/마감해제 (1차)

- **목적**: 마감을 검증·승인 워크플로우로 구현하는 결산 관리 화면·엔진. `FiscalYear`(기수·시작/종료일·상태)+`FiscalPeriod`(월별·상태), 회사별 회계달력(12월말/3월말 등) 지원(§11.9-(1)).
- **기간 상태머신**(§11.9-(1), §12.2):

```
FUTURE(미개시) → OPEN(입력 가능) → TEMP_CLOSED(가마감) → CLOSED(본마감) → LOCKED(신고 완료·아카이브, 해제 불가)
                                        ▲                     │
                                        └── REOPENED ◀── REOPEN_REQUESTED (마감해제 승인 워크플로우)
```

- **마감 전 검증 체크리스트(`ClosingChecklist`)** — 가마감 게이트(§11.9-(2)):

| 검증 항목 | 처리 | 근거 |
|---|---|---|
| 미결/반려/미승인 전표 존재 | 목록 제시, 승인 또는 차기 이월 처리 전 가마감 차단 | §11.9-(2) |
| 시산표 차대 불일치 | 마감 차단(원천 오류 수정 필요) | §11.9-(2) |
| 차원 필수값 미입력 전표 | §9 규칙 기준 보완 목록 제시 | §11.9-(2) |
| 외부수집/변환 스테이징 미반영 잔여 건 | 반영 또는 차기 이월 확인 | §11.9-(2), §8.4·§11.6 |
| 은행/카드 잔액 대사 미완료 | 경고(정책상 차단 선택 가능) | §11.9-(2), §11.11 |
| 가마감 후 신규 입력 | 가마감 기간 입력 차단(권한 예외 시 사유·로그) | §11.9-(2) |
| (HYBRID) 분담 영역별 입력 완료 | 양측 분담 영역(§21.6) 입력 완료 확인 — 미완료 측 잔여 목록 | §11.9-(2) |
| 계정별 명세 미tie-out(diff≠0) | 마감 차단 항목으로 노출(`verified=false`) | §11.14-(2) |
| 미해결 미결항목(증빙불비 등) | 마감 게이트 차단/경고 | §15, §18.3 |
| NTS `REJECTED`/미수신 전자세금계산서 | 체크리스트 표시, 필수 거래는 마감 차단 가능 | §11.4-(1), §15 |

- **마감·마감해제 절차**(§11.9-(3)):

| 단계 | 동작 | 근거 |
|---|---|---|
| ① 가마감(`TEMP_CLOSED`) | 체크리스트 통과 후 주기장 측 담당자 실행. 분산 락 `lock:tenant:{id}:closing:{yyyymm}` 획득 + 상태 재검증 | §11.9-(3)1 |
| ② 본마감(`CLOSED`) | 주기장 측 결재(마감 권한자, **SOD: 전표 작성자와 분리**) → 마감 스냅샷(`TenantSnapshot`) 자동 생성 → 마감 잠금 인터셉터 활성 | §11.9-(3)2 |
| ③ 이용회사 확인(Ack) | 주기장=OPERATOR면 본마감 후 마감 결과 요약 송부 → `CONFIRMED`/`OBJECTED`(사유). `ackPolicy=REQUIRED`면 확인 전 신고 생성 차단, `OBJECTED`는 보완 후 재확인 | §11.9-(3)3, §21.4 |

| 간접법 | 당기순이익 → 비현금 항목 가감(감가상각비·대손상각·충당금전입·외화환산손익·유형자산처분손익 등) → 운전자본 증감(매출채권·재고자산·매입채무·선급/선수 기초·기말 비교) → 영업활동 CF. 투자/재무는 관련 계정 증감·거래 분류. Smart A 계산식: I.영업=당기순이익+현금유출없는비용−현금유입없는수익+자산·부채변동, II.투자=유입−유출, III.재무=유입−유출, IV=I+II+III, IV+V.기초현금 = VI.기말현금(§25.9.8) |
| 직접법 | 현금및현금성자산 계정(`CashEquivalentAccount` 정의)의 상대계정·거래유형 분석으로 유입(매출·이자수취)·유출(매입·인건비·이자지급) 항목별 직접 집계 |

- **매핑·분석 원칙**: `CashFlowMapping`(계정/거래유형→활동·세부항목, 표준은 공통 표준 6장 배포+회사 확장). **미매핑 계정·현금 거래는 자동 추정 금지 → 검토대상 표시**. 매출채권은 충당금 차감 순액 분석, 투자·재무 동일 계정 중복 시 실제 발생 쪽만, 기초·기말현금 = 제출용 재무상태표 현금및현금등가물, 단기차입금 총액 기재 원칙(빈번 시 순증감), 비현금 거래(현물출자·전환)는 제외·주석 표시(§11.3.1-(2)(3), §25.9.8).
- **정합성 검증·조정 워크플로우**(§11.3.1-(4)(5)):

| 단계 | 동작 |
|---|---|
| 현금 정합성 검증 | 기초현금+영업·투자·재무 순현금흐름+환율변동효과 = 기말현금, 기말현금 = 재무상태표 현금및현금성자산 잔액 일치 |
| ① 검증 경고 발령 | 오차 발생 시 결산 화면에 붉은색 경고 + 불일치 금액(`diffAmount`) 출력(교착 방지 보완 프로세스) |
| ② 수기 매핑 조정 툴 | 활동 분류 누락 전표(차·대 모두 대체계정 등)를 찾아 수동으로 현금흐름 차원을 매핑하는 원클릭 그리드 팝업 |
| ③ 조정 데이터 발행 | 추적 불가 소액 오차는 **승인 사유 입력 후** 특정 활동(영업활동기타 등) 귀속. **G/L 미변경 — 출력용 차원 테이블(`CashFlowAdjustment`/`CashFlowMapping`)에만 기록** + `DataChangeLog` 감사 |
| 재검증·명시 | 조정 후 정합성 재수행, 조정 내역은 결산 산출물에 명시. 회계 장부 자동수정 금지 |

- **산출물**: 영업/투자/재무활동 현금흐름표(비교식 초기이월 입력 가능). 마감/확정 시 스냅샷 보관(`TenantSnapshot`), 생성 이력·매핑버전 감사(§11.3.1-(4)).
- **연계**: 제출용 재무상태표(SA-CLS-03) 데이터 기반, 선행 SA-CLS-03·04(§25.9.8).

### 4.22 SA-CLS-20 중단사업손익계산서 (2차)

- **목적**: 중단사업 손익 항목 조회/출력 + 손익대체 워크플로우. [환경설정] 중소기업 회계기준 "0.사용안함" 시 사용 가능, "중단사업코드설정"(부서 또는 프로젝트 중 하나) 데이터만 반영(§25.10.6).
- **조회조건**: 일자(조회 월), 구분(0.과목/1.세목), 내역(중단사업코드 설정 부서/프로젝트별 손익항목), 더보기(중단사업설정(부서/프로젝트), 법인세 효과(법인세율 입력 후 계산·수정 가능), 계정과목코드보기, 원장조회(전표수정))(§25.10.6).
- **기능**: 전표추가 — 중단사업 손익대체분개를 [일반전표입력]에 전송. **이전 분개 금액에서 현재 조회 금액을 차감한 차액만 추가(이중 반영 방지), 조회일 말일자 전송**. BK는 중단사업 코드를 6차원 중 DEPARTMENT/PROJECT 차원으로 매핑해 필터 집계하고, 손익대체분개를 `CLOSING_DR`/`CLOSING_CR` 라인으로 차액 기반 생성. 법인세 효과 = 세율 파라미터(§25.10.6).
- **산출물**: 중단사업 손익현황(법인세 효과 포함), 손익대체분개 전표. **연계**: [환경설정] 중단사업코드(P01), SA-CLS-04 중단사업 유형설정 연동(§25.10.6).

### 4.23 SA-CLS-21 중단사업원가명세서 (2차)

- **목적**: 중단사업 원가명세서 작성(관리용) + 원가대체 워크플로우. 사용 조건은 SA-CLS-20과 동일(중소기업 회계기준 미적용+중단사업코드 설정)(§25.10.7).
- **조회조건**: 일자(조회 월), 구분(원가설정 코드), 새로 불러오기(**'확인' 시 중단사업제조원가 안분 분개도 삭제**), 전표추가(원가대체분개 [일반전표입력] 전송 — 이전 전표추가분과의 차액만, 말일자), 더보기(원가설정(매출원가·원가경비코드), 중단사업설정, 원장조회, 계정과목코드보기)(§25.10.7).
- **기능**: BK는 중단사업 코드(부서/프로젝트)를 코스트센터/프로젝트 차원으로 필터해 원가 안분하고, 원가대체분개를 `CLOSING_DR`/`CLOSING_CR` 라인으로 차액 기반 생성, 원가설정을 차원 기반 원가 매핑으로 구현(§25.10.7).
- **산출물**: 중단사업 원가현황(관리용), 원가대체분개 전표. **연계**: SA-CLS-05·SA-CLS-20 연동, [환경설정] 원가설정(§25.10.7).

### 4.24 [BK 공통] 보고패키지 · 계정별 명세 tie-out · 계산시트 (1차)

- **보고패키지(`ReportPackage`)**: 정해진 목차로 매월 조립·검증되어 이용회사에 전달되는 패키지. 템플릿(`ReportPackageTemplate`)은 관리회사가 공통 표준(`STD_REPORT_PACKAGE`)으로 배포 — 구성요소 목록·순서·필수여부·기장모드/요금제·출력 언어·산출 형식(xlsx/pdf)(§11.13-(1)).
- **구성요소 카탈로그(`ReportComponent`)**: BS/IS(PL)/TB/현금흐름표/자본변동표/분개장(JE)/총계정원장(GL)/계정별 명세서/고정자산명세/채권·채무 aging/부가세 집계/매입원장 세무상태/원천세/관계사 잔액/제조원가명세서(CM). 결산 스냅샷·원장에서 자동 생성, 회계기준별·언어별 산출(§11.13-(2)).
- **상태머신·생성 흐름**(§11.13-(3)):

```
PENDING → GENERATED → VALIDATED → REVIEWED → DELIVERED → ACKED
정기 스케줄(월마감 후) → 컴포넌트 자동 생성 → 패키지 조립 → 정합성 검증(tie-out 전건 통과 +
미결항목 0건) → 주기장 측 검토·결재 → 이용회사 전달(공유링크/이메일) → 이용회사 확인(Ack)
```

- **버전·재생성**: 재무제표 변경 시 패키지 자동 무효화·재조립 안내, 전달본은 파일해시·표준/taxonomy 버전·생성자 기록(§11.13-(4)).
- **계정별 명세·tie-out**(§11.14): `AccountSchedule` 명세 라인(거래처/항목·적요·금액·증빙참조·비고), 부채성·충당성 항목은 이월(Opening)/사용(Utilization)/설정(Accruals)/대체(Reclass)/기말(Closing) movement 컬럼. `ScheduleTieOut` — 명세 합계 = GL/TB 마감잔액 자동 대조 → `diff` 계산 → `verified(boolean)`. **미tie-out 계정은 마감 체크리스트·보고패키지 검증 차단 항목**. 명세 소스 = (a)보조원장/거래처원장 자동집계(P04) (b)스케줄형 항목(선급/충당/리스 §11.15) (c)수기 명세+증빙 연결. 전월 명세 carry-forward(movement만 갱신).
- **계산시트(M8)**: 대손충당금·감가상각·충당금·외화평가 등 산출 근거를 셀 단위 계산식이 보존되는 `CalculationSheet`로 자동 생성, 명세·전표·원장과 양방향 drill-down. 근거자료 첨부(증빙 프레임워크 재사용). 계산시트 변경 → 연결 주석(`DisclosureNote`) 재생성 대상 표시 + 보고패키지 무효화·재검증(§11.14-(6)).
- **주석 자동화(N10)**: 매출채권 및 기타채권(Invoice/aging)·대손충당금(설정·환입 Roll-forward)·Aging 분석(`AgingSnapshot`)·Roll-forward(기초→증감→기말: 유형·무형자산·충당부채·충당금)·금융상품·유형/무형자산·충당부채를 원장·보조부에서 자동 산출. 재무제표 수치 ↔ 주석 수치 자동 교차 검증(불일치 시 검토대상), 주석별 근거 파일·산출 로직 링크로 Audit Trail 확보. 자동화 MVP 범위는 §23-54 확정(§11.3.2-(4)).
