## 1. 페이즈 개요

- **목표**: P06 부가가치세신고서(SA-VAT-01)의 첨부·부속서류 중 ① 매입세액 통제 서식(불공제 명세·안분/정산/재계산, 감가상각자산 취득, 대손세액), ② 멀티사업장 신고 서식(총괄납부 사업장별 명세, 사업자단위과세 명세), ③ 사후 신고 서식(과표수정·추가납부계산서, 경정청구서), ④ 영세율·환급 부속 2종(관세환급금, 외국인관광객 면세판매·환급)과 ⑤ 업종별 2차 서식 7종(면세유류~미용성형 의료용역)을 구현한다(§25.13, §25.12.8~13, §25.14.5~7).
- **범위 요약**: 1차 9메뉴 + 2차 7메뉴, 총 16메뉴. 메뉴 채널은 TN-09 부가세·신고서류(§26.1-(4): SA-VAT 50화면 배치 중 본 페이즈 16화면).
- **선행조건**:
  - P03 매입매출전표(SA-JNL-02): 54.불공 + 불공제사유 코드(51→54 변경 시 불공제사유 팝업, 54→전송 시 본 명세서 자동반영 — §25.4.2.2), 고정자산 계정 분개 분기(51/52/54/55/57 + 고정자산 계정), 적요 06/07 트리거(§11.6.2-(2) 부가세 통제 메타).
  - P06 SA-VAT-01 신고서 마감 상태머신·수정신고 차수(정기신고 마감 선행), 추가 차가감납부세액 ≥0/<0 분기(§25.11.1) — SA-VAT-27/28의 판정 원천.
  - P06 SA-VAT-07 전자신고 파이프라인(제작→검증→제출) — 본 페이즈 서식은 첨부서식으로 전산매체에 수록(§25.11.7 "모든 첨부서식 SA-VAT-03~50" 연계).
  - P01 회사등록 본지점설정(본점·총괄납부/사업자단위과세, §25.1.1)·국세환급금계좌·세무대리인 정보, 거래처등록 카드탭 가맹점번호(§25.12.11).
- **MVP 연관**: MVP 최우선(P03 전표·P01 계정과목, §22 N13)의 후속 소비처. 카드 VAT 공제 자동 분석(N11, §11.1.1-(16))은 기능 범위 확장을 통제하며 단계적으로 반영하는 항목으로, 본 페이즈 SA-VAT-22 불공제 판정과 연계된다(§22.2 N13).

## 4. 화면별 상세설계

### 4.1 공제받지못할매입세액명세서 `SA-VAT-22` (1차, §25.13.1)

- **목적**: 매입세액 불공제 대상 내역을 작성하는 부속서류(54.불공 + 불공제사유 자동반영, 직접입력 가능) — BK **공통매입세액 안분·정산·재계산 엔진**으로 생성. 겸영(과세/면세) 공급가액 비율·감가상각자산 체감률 룰엔진, 불공제 사유 코드 카탈로그를 전표 증빙과 연동하고 사업장 차원으로 분기한다.
- **진입경로**: TN-09 부가세·신고서류 > 부가가치세 첨부서류(§26.1).

## 6. API 설계

> P06 §6의 부속서류 공통 API(`/attachments/{formType}/load`·`/close`)를 재사용하고, 본 페이즈 특화 리소스만 추가한다. 엔드포인트는 §25.13~25.14의 화면 기능(불러오기·마감·수정차수·정산 계산)을 REST로 구체화한 대표 설계다(§13.1 공통 인터셉터 준용).

| 메서드 | 엔드포인트 | 설명 | 멱등성·오류 |
|---|---|---|---|
| POST | `/api/tenant/vat/attachments/{formType}/load` | 부속서류 불러오기(계정/거래처/가맹점 설정 기반, 기존 삭제 후 재집계 또는 조건별 삭제 — SA-VAT-21) | Idempotency-Key |
| POST | `/api/tenant/vat/attachments/{formType}/close` | 부속서류 마감(SA-VAT-18: 수량·사업자/주민번호 필수, 마감 후 수정불가) | `VALIDATION_FAILED` |
| POST | `/api/tenant/vat/non-deductible/{id}/apportion` | 공통매입세액 안분 계산(계산식 1~5, 예정) | 자동계산 결과 반환 |
| POST | `/api/tenant/vat/non-deductible/{id}/settle` | 공통매입세액 정산(확정 — 면세확정비율·기 불공제 차감) | 〃 |
| POST | `/api/tenant/vat/non-deductible/{id}/recalculate` | 납부/환급세액 재계산(체감률 5%/25%·경과 과세기간·면세비율 증감 ≥5%) | 〃 |
| GET | `/api/tenant/vat/bad-debts?period&tabType` | 대손발생/변제 명세 조회 | — |
| POST | `/api/tenant/vat/bad-debts` | 대손 건 등록(10/110 자동, 10년 기간·부호 규칙 검증) | `VALIDATION_FAILED` |
| POST | `/api/tenant/vat/branch-statements/{type}/load` | 사업장별/단위과세 "부가가치세 불러오기"(본·지점 신고서 반영, 기존 삭제) | Idempotency-Key, 지점 권한 `FORBIDDEN_SCOPE` |
| POST | `/api/tenant/vat/returns/{id}/amend-forms` | 수정차수 연동 서식 생성(≥0 → 추가납부계산서 / <0 → 경정청구서 자동 분기) | 수정신고서 저장 선행 검증 |
| PUT | `/api/tenant/vat/amend-forms/{id}` | 수정사유·경정청구이유·환급계좌 보정(이유 1/2 중복 차단) | version 낙관적 락 |
| POST | `/api/tenant/vat/card-rules/evaluate` | 카드 VAT 공제/불공제/검토필요 자동 판정(N11) | 검토 큐 라우팅, reason code 필수 |
| PUT | `/api/tenant/vat/card-rules` | 고객사별 Rule 설정·반복 거래 보정 | `VALIDATION_FAILED` |
| POST | `/api/tenant/vat/efiling-credit/load` | 수임업체 건수 집계(직전과세기간 마감분 × 1만원, 한도 검증) | 한도 초과 시 경고 |
| POST | `/api/tenant/vat/attachments/{formType}/excel-import` | 엑셀 업로드(SA-VAT-33/34/35/20 등) | 오류행 리포트 |

- **오류코드**(§13.3): `VALIDATION_FAILED`, `CLOSING_LOCKED`(마감 후 수정), `PERIOD_LOCKED`, `NOT_PRIMARY_BOOKKEEPER`, `FORBIDDEN_SCOPE`(지점 권한 없음), `IDEMPOTENCY_CONFLICT`(409), `EXTERNAL_INTEGRATION_FAILED`.
- **Scope**(§13.3): 전자신고 수록·마감은 위험 scope로 Step-up, 지점 데이터는 사업장 차원 RLS로 접근 제어(§25.13.4).

## 11. 완료 기준(DoD)·테스트 관점

**DoD**

1. 1차 9메뉴(SA-VAT-22~28·33·34)가 P03 전표(불공제사유·고정자산 계정·영세 계열)에서 자동 집계 → 보정 → 마감 → SA-VAT-01 란 연동 → SA-VAT-07 전산매체 수록까지 E2E 동작(§25.13, §25.11.7).
2. 공통매입세액 안분(계산식 5종)·정산·재계산(체감률 5%/25%) 3단계 룰엔진이 자동 계산되고 마감 검증(면세공급가액=0 오류)이 동작(§25.13.1).
3. 대손 부호 규칙·10년 공제기간·10/110 자동계산과 신고서 8란/47란 부호 반영이 정확(§25.13.3).
4. 총괄납부·사업자단위과세 본·지점 불러오기 합산과 지점 권한(RLS) 격리, 직접 조정 항목(대손·미환급·고지) 통제가 동작(§25.13.4~5, §11.4).
5. 수정신고 ≥0/<0 자동 분기로 SA-VAT-27/28이 생성되고, 경정청구이유 중복 금지·신고인정보 수정불가·승인 게이트(`filingApprovalRequired`)가 동작(§25.13.6~7, §21.4).
6. 검증 표(8절) 30항목이 서버/클라 이중 검증으로 구현(00_개요 §3 UI 공통).
