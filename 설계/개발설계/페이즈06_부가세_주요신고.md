# BK 개발 상세설계서 — 페이즈 06. 부가세 주요신고 (SA-VAT-01~15)

- 기준 문서: `bk_설계서_v1.1.md` (§25.11 부가가치세 주요신고서류, §25.12 업종별 첨부서류, §11.4 매입매출/부가세, §21.4 신고 책임, §22.2 개발 차수 계획)
- 작성일: 2026-07-03
- 개발 차수: **1차(2609~2611) = SA-VAT-01~14** / **2차 = SA-VAT-15** (§22.2 페이즈 06 행, 00_개요 §1)
- 선행 페이즈: **P03 전표·자동전표(매입매출전표 SA-JNL-02 과세유형 11~64·증빙, SA-ATX 수집자료)** — 부가세 신고는 P03 매입매출·증빙 데이터를 전제(00_개요 §2). P01 회사등록(과세유형·본지점·환급계좌)·거래처등록(사업자번호) 선행.
- 공통 설계 표준(멀티테넌시·감사필드·오류코드·권한·멱등성·UI·기술스택)은 `00_개요_페이즈일정.md` §3을 따르며 본 문서에서 반복하지 않는다.

## 1. 페이즈 개요

- **목표**: 매입매출전표(P03)의 과세유형(매출 11~25·매입 51~64) 자동 집계 위에 ① 부가가치세신고서 생성·마감 상태머신, ② 세무리스크 자동검증(MRI) 사전검증, ③ 4대 합계표·명세서(세금계산서·신용카드 수령/발행·계산서), ④ 전자신고 파이프라인(제작→검증→제출)·납부서, ⑤ 업종별 부속서류 6종(부동산임대~현금매출)을 구현한다(§25.11~25.12).
- **범위 요약**: 1차 14메뉴(SA-VAT-01~14) + 2차 1메뉴(SA-VAT-15). 메뉴 채널은 TN-09 부가세·신고서류(§26.1-(4): SA-VAT 50화면 배치 중 본 페이즈 15화면).
- **선행조건**:
  - P03 매입매출전표입력(SA-JNL-02)의 2계층 구조(세무헤더+분개라인)와 `tax_type_master`(과세유형 단일 진실원) 가동(§0.4 P1·P2, §25.4.2.2).
  - P03 SA-ATX-02~05 전자세금계산서/계산서/카드/현금영수증 수집·자동전표(전자/전자외 판정 원천, §25.5).
  - P01 회사등록(법인/개인·과세유형·본지점설정·국세환급금계좌, §25.1.1)·환경설정(과세유형 추가 25.매세/63.복지, 봉사료, 공급가액 소수점 끝수처리, 중고차량/차대번호 — §25.1.3 회계탭 ③④⑤⑥)·거래처등록(사업자번호·단위신고거래처, §25.1.6).
  - P05 마감 워크플로우: 신고 완료 기간 `LOCKED` 전환·마감해제 불가(§11.9-(1)(3)).
- **MVP 연관**: MVP 최우선은 P03 전표·P01 계정과목(§22 N13)이며, 본 페이즈는 그 산출 데이터(과세유형·증빙)의 첫 번째 법정 신고 소비처다. 신고서 생성·전자신고 전송은 **주기장 측 수행**이 원칙(§11.4, §21.4).

## 3. 업무 흐름 (End-to-End)

```
[P03] 매입매출전표(과세유형 11~64) + SA-ATX 수집자료 + 증빙
   │  과세유형 → 신고서식 라우팅(§25.4.2.2: 유형 자체/계정 기반/적요 기반 분기 3종)
   ▼
① 부속서류·합계표 작성 (SA-VAT-03~06, 09~14) — 불러오기(전표 자동집계) → 보정 → 마감(매출·매입 동시)
   ▼
② 부가가치세신고서 생성 (SA-VAT-01) — 신고구분(예정/확정/영세율조기환급/기한후)·사업장 과세단위 선택 → 자동집계 → 직접입력 항목 보정
   ▼
③ 신고서 마감 상태머신 (§25.11.1) — 마감 → 첨부서식 조회·전자신고여부(0/x) 지정 → MRI 60여 항목 검사(SA-VAT-02)
   │  오류리스트 <유의사항>은 강제마감 가능
   ▼
④ (주기장=OPERATOR) 이용회사 확인(Ack)·신고 승인 (§21.4) — filingApprovalRequired=Y(기본): 신고서 요약(과세표준·세액) 송부 → Step-up 승인
   ▼
⑤ 전자신고 (SA-VAT-07) — 제출자등록 → 전산매체 제작(.101/.102, 비밀번호 8~15자리) → 검증 → 홈택스 제출(약 15단계 자동화)
   │  신고상태: 제작오류/제작완료/검증오류/검증완료/제출오류/제출확인
   ▼
⑥ 납부서 출력 (SA-VAT-08, 2D 바코드) + 신고 완료 기간 LOCKED 전환(§11.9) + 수정신고/경정청구 분기(추가 차가감납부세액 ≥0 / <0, §25.11.1 → P07 SA-VAT-27/28)
```

- 신고유형은 예정/확정/수정/기한후 전체 + 가산세를 지원하고, 사업장 차원과 연계한 사업장별 신고/사업자단위과세 합산을 수행한다(§11.4).
- 확정신고 시 예정신고누락분(1기 4~6월/2기 10~12월 반영)·예정신고미환급세액·예정고지세액을 처리한다(§25.11.1).
- 매입원장 세무구분(`taxCategory`)·국세청 전송상태(`ntsStatus`)가 신고 집계의 원천이며 홈택스 수신 결과와 주기적 대사한다(§11.4-(1)).

## 4. 화면별 상세설계

### 4.1 부가가치세신고서 `SA-VAT-01` (1차, §25.11.1)

- **목적**: 일반·간이과세 사업자의 6개월 과세기간(중간 예정신고기간 포함) 부가세 신고서를 신고서 엔진으로 자동 생성. 정기/수정신고 차수 관리(정기신고 저장·마감 후 동일기간 수정신고 가능). 음식·숙박·기타서비스업은 사업장현황명세서 첨부.
- **진입경로**: TN-09 부가세·신고서류 > 주요 신고(§26.1).

## 6. API 설계

> `POST /api/tenant/filings/{id}/approve`는 §13.2 명시 엔드포인트, 나머지는 §25.11의 기능(부가세 집계 API·마감·제작·검증·제출)을 REST로 구체화한 대표 설계다. `VatService`·`TaxInvoiceService`는 §13.1(부록 B 14.1·부록 C 15장 준용 + 주기장 권한 검증 공통 인터셉터).

| 메서드 | 엔드포인트 | 설명 | 멱등성·오류 |
|---|---|---|---|
| GET | `/api/tenant/vat/returns?fiscalYear&period&filingType&taxUnit&amendmentSeq` | 신고서 조회(작성조회: 마감여부·과세구분) | — |
| POST | `/api/tenant/vat/returns` | 신고서 생성(과세유형 11~64 자동집계 — 부가세 집계 API, §25.11.1) | Idempotency-Key |
| PUT | `/api/tenant/vat/returns/{id}` | 직접입력 항목 보정(가산세·예정고지 등) | version 낙관적 락 |

- **Scope**(§13.3): `tenant:filing:approve`(이용회사 승인), 신고 전송·마감해제·평문 조회 등 위험 scope는 Step-up 인증 필수.

## 11. 완료 기준(DoD)·테스트 관점

**DoD**

1. 1차 14메뉴(SA-VAT-01~14)가 과세유형 11~64 전표에서 자동 집계·마감·전자신고 파일(.101/.102) 제작·홈택스 제출까지 E2E 동작(§25.11.1·7).
2. 신고유형 4종(예정/확정/영세율조기환급/기한후)+수정신고 차수·경정 분기(≥0/<0)와 사업장별/총괄/사업자단위 과세단위가 동작(§11.4, §25.11.1).
3. MRI 60여 항목 검사·<유의사항> 강제마감·역추적 대사가 동작(§25.11.1~2).
4. `filingApprovalRequired`·Ack·주기장 인터셉터·`LOCKED` 전환의 통제 시나리오 통과(§21.4, §11.9).
5. 검증 표(8절) 22항목이 서버/클라 이중 검증으로 구현(00_개요 §3 UI 공통).
