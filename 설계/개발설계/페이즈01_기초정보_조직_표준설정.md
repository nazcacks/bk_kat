## 1. 페이즈 개요

### 1.1 목표

기장·관리 대상 회사(테넌트)의 **마스터 기초정보 전체**를 구축하는 페이즈. 회사등록(테넌트+회계기간), 권한, 환경설정(코드체계·6 관리차원 스위치), 조직(부서·사원), 거래처, 계정과목(COA 3기준), 현장·프로젝트·업무용승용차, 코드변환·마감후이월까지 — 이후 모든 페이즈(전기이월 P02, 전표 P03, 장부 P04, 결산 P05, 부가세 P06~08)의 외래키 마스터와 검증 메타데이터를 공급한다(§25.1).

### 1.2 MVP 연관

- **계정과목 관리(WJV-08/09, §11.6.2)는 일반전표입력(WJV-01)과 함께 전 플랫폼 MVP 최우선 대상**이다(§22 v1.1 rev N13, 2026-07-02 회의 결정 1·2·10). 본 페이즈 내 착수 순서에서 SA-BAS-07(계정과목)을 최우선으로 한다.
- R18 신규 화면 2종(계정 구조설계·FS 매핑관리, §11.6.2-(4))은 개발요청 ★★★ 필수 항목으로 1차에 포함한다(§22.2).

### 1.3 선행조건

- 신규 온보딩 게이트 ①~⑧단계(관리회사 준비→표준 배포→테넌트 생성→구독→기장 모드→사용자·인증→역할·결재선→표준 채택·차원 설정)가 본 페이즈 화면군의 운영 전제다(§22 온보딩 절차). 본 페이즈는 ⑨단계 "기준정보 적재"의 기준정보 부분을 담당하고, 개시잔액 적재·검증 게이트는 페이즈 02가 담당한다.
- 표준 카탈로그(`STD_ACCOUNT`·`STD_VAT`·`STD_ROLE` 등, §6.1)의 배포·채택 메커니즘(§6.3)이 회사별 COA 생성·권한 템플릿의 원천이다.

### 1.4 범위 요약

## 4. 화면별 상세설계

### 4.1 SA-BAS-01 회사등록 (§25.1.1)

- **목적**: 기장·관리 대상 사업장(테넌트) 마스터 최선행 등록. 부가세·법인세·종소세·원천징수·연말정산 신고와 (전자)세금계산서·재무제표 출력의 단일 기준. Smart A "기수 단일파일" 모델을 `tenant` + `fiscal_period`(복수 기수) 모델로 치환(§25.0.2 C1).
- **진입경로**: 기준정보 › 기초정보관리 › 회사등록.
- **조회조건**: 회사 목록(관리회사 권한 시 담당 테넌트 목록), 기수 선택.

**입력 필드 — 기본사항·기본정보 탭 (§25.1.1)**

|---|---|---|---|
| ① | 기수 | int(개업연도=1기 자동계산) | tenant 하위 fiscal_period seq 관리. 최초 등록 후 기수↔회계기간 정합 불일치 시 수정 불가 → 등록 시 정확 입력 강제 |

- **검증**: 사업자등록번호 오류 시 전자세금계산서·전자신고 차단 / 기수·회계기간 등록 후 수정 불가 가드 / 저장 미클릭 시 미반영 경고 / 사업자번호 활성 테넌트 중복 = 테넌트 등록 저장 차단(§15).

- **연계**: 모든 후행 화면의 부모 엔티티. 본지점·과세단위 → 사업장 차원(§9). 외부: WEHAGO 회사설정 동기화 API, 국세청 전자신고 파이프라인.
- **권한**: 회사 관리자·`operator:masterdata:write`(§13.3). 기수 정보는 SA-DAT-07 기수변경(P10)과 공유.

### 4.2 SA-BAS-02 권한설정 (§25.1.2)

- **목적**: 회사관리자가 회계 서비스 관리자를 지정하고 일반사용자에게 모듈·메뉴별 권한을 배포. 사용자×테넌트×메뉴 매트릭스를 RBAC(§4.3) + RLS와 결합해 행 수준 접근 통제.
- **진입경로**: 기준정보 › 기초정보관리 › 권한설정.

**입력 필드**

| # | 필드 | 값 | 의미 (§25.1.2) |
|---|---|---|---|
| ① | 사원 | 배포 사원 자동반영 | 회사설정>서비스관리>서비스별 배포관리 연동 — user↔tenant 매핑 |
| ② | 회사권한 | 0.전체회사/1.관리회사 | 0=대표+관리회사 전체 접속, 1=등록 회사만 → **RLS 테넌트 스코프 범위 결정** |
| ③ | 서비스명 | 구매 서비스 표시 | read-only |
| ④ | 사용여부 | 0.여/1.부 | 부 시 모듈 접속 불가 |
| ⑤ | 메뉴사용 | 1.전체메뉴/0.선택메뉴 | 선택 시 메뉴별 개별 권한 = 메뉴 권한 비트맵(`RoleMenuPermission`+`MenuActionGrant`, §12.1·§4.3) |
| ⑥ | 모듈 라이센스 현황 | read-only | 회계/급여/물류 — 서비스마켓 바로가기 |

- **기능**: 메뉴별 권한 부여/회수, 사용여부 토글. API 게이트웨이 인가 미들웨어가 사용여부·메뉴사용을 enforce(§25.1.2).

- **기능**: 마감(확인 메시지 후 진행, 완료 시 "마감" 표시), 마감취소.
- **검증**: 이월 전 재무제표(제조원가명세서·손익계산서·이익잉여금처분계산서·재무상태표·중단사업손익/원가=종료월, 기간별손익/원가=시작~종료월, 프로젝트 손익/원가=프로젝트별 종료월)를 과목별/세목별/관리용/제출용/표준(법인용) 탭별로 조회 후 마감해야 정확 이월 / 동일 회사 다른 사용자 미사용 필요(분산 락 — §15 "분산 락 미획득 = 비즈니스 예외") / 계정잔액 0원이면 거래처 잔액 미이월 / 이월기수 일반·매입매출전표 기존 데이터는 마감 시 미삭제 / 모든 체크 해제 후 마감 = 이월 없이 마감만·특정 메뉴만 체크 = 해당 데이터만 이월 / 마감후이월 미실행 시 기초데이터 없이 작업 가능(§25.1.14).
- **산출물**: 이월기수 전기분 재무상태표·손익계산서·원가명세서·이익잉여금처분계산서, 거래처별/부서사원별/현장별/외주처별/프로젝트 초기이월, 고정자산·업무용승용차.

## 11. 완료 기준(DoD) · 테스트 관점

1. **MVP 게이트**: SA-BAS-07이 3기준 트리 조회·계정 CRUD(사용중지)·`coa_control` 설정·법인세표준계정 매핑·적요 등록을 제공하고, P03 일반전표입력이 `coa_control` 기반 검증(`DIMENSION_REQUIRED` 등)을 소비할 수 있다(§11.6.2, §22 N13).
2. 회사등록으로 법인/개인 각 1개 테넌트를 생성하고 기수·회계기간 정합 가드, 사업자번호 체크섬·중복 차단이 동작한다(§25.1.1, §15).
3. 환경설정의 관리코드 스위치가 `DimensionConfig`로 저장되어 차원 마스터(04/05/08/09/10) 노출·전표 검증에 반영되고, 변경 영향 분석이 산출된다(§9.3, §10.2).
4. 거래처 3탭(일반/금융/카드) 전 필드 입력·엑셀 일괄등록·진위확인·승인 워크플로우(TEMP→PENDING_APPROVAL→ACTIVE)·R8 6종 통제 판정 기준이 동작한다(§25.1.6, §11.1.1-(8)).
5. R18 2화면: 구조설계에서 코드 길이·`coa_control` 매트릭스 편집, FS 매핑관리에서 4종 버전 관리·버전 복사·적용기간·미매핑/중복매핑 자동 검출이 동작한다(§11.6.2-(4)).
6. 코드변환: 가드 6종(자금항목·마감·적요·자동전표 출처·품목·기간) 통과/차단 케이스와 멱등 재실행 검증(§25.1.13).
7. 마감후이월: 체크박스 조합별 선택 이월, idempotent 재이월(기존 삭제 후 재적재), 분산 락, 차량 동일성 판정, 잔액 0원 스킵 — P02 화면에서 적재 결과 조회 확인(§25.1.14).
8. 비기능: RLS 테넌트 격리(교차 조회 0건), PII 암호화·마스킹, `DataChangeLog` 전후값, 표준 오류코드 응답(00_개요 3절).
9. 테스트 관점: 법인/개인·과세유형별 분기, 세목 사용/미사용(3/5자리)·가변(8/9자리) 코드 체계, 외국법인 GAAP 추가(WJV-09D 4종 초기화), WEHAGO 미연동 회사(연동 무영향 확인), 기장 모드 3종별 메뉴 접근.
