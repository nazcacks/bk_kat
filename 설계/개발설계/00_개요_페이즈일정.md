## 2. 페이즈 의존관계

```
P01 기초정보(회사·권한·환경·COA·거래처·차원) ─┬─▶ P02 전기이월 ─▶ P03 전표 ─▶ P04 장부·보조부 ─▶ P05 결산
                                              │                    │                              ├─▶ P06·07·08 부가세·고정자산
                                              │                    └─(자금 관리항목)──────────────▶ P09 자금·예산·어음(2차)
                                              └─▶ P10 데이터관리(백업·이관·기수 — 전 페이즈 횡단)
```

- P03은 P01(계정과목·거래처·차원·환경설정)과 P02(개시잔액) 완료를 전제로 하되, P02와 병행 개발 가능(개시잔액 검증 게이트만 선행).
- P05 결산은 P03 전표·P04 장부 집계를 전제. P06~08 부가세·고정자산은 P03 매입매출·증빙 데이터를 전제.
- P09는 P03 전표의 자금 관리항목(WJV-04~06) 데이터 위에 보조부·현황을 구축(전부 2차).

## 3. 공통 설계 표준 (전 페이즈 적용 — 각 문서에서 반복하지 않음)

| 항목 | 기준 | 설계서 근거 |
|---|---|---|

| 감사 필드 | `created_at/by, updated_at/by, version(낙관적 락), source` 기본. 회계 데이터 물리 삭제 금지(취소·역분개) | §12.3, §16.1 |
| 변경이력 | CUD·승인·마감·전송은 `DataChangeLog`(전후값·IP·행위자) append-only | §17 |
| 오류코드 | §13.3 표준(`VALIDATION_FAILED`~`ASSIGNMENT_REQUIRED`, 전표 도메인 `AUX_REQUIRED`/`DIMENSION_REQUIRED`/`POSTED_IMMUTABLE`/`QUICK_ENTRY_INVALID_ACCOUNT`/`NUMBERING_CONFLICT`) + 운영 콘솔 §26.4 | §13.3, §26.4 |
| 권한 | scope 표준(§13.3), 메뉴 권한 `RoleMenuPermission`+`MenuActionGrant`(미노출 원칙), SOD(작성≠승인), Step-up(마감해제·신고 전송·평문 조회) | §4.3, §4.5, §3.2 |

| 멱등성 | 생성·전송·반영 API `Idempotency-Key`, 배치 멱등·재처리 | §13.3, §14 |
