## 1. 페이즈 개요

### 1.1 목표

신규·이관 테넌트의 **전기(전전기·전전전기 포함) 재무제표 스냅샷과 계정×차원별 개시잔액(opening balance)** 을 구축하는 페이즈. 전기분 재무상태표·손익계산서·원가명세서·이익잉여금처분·자본변동표·현금흐름표(SA-OPN-01~06)가 비교식 재무제표의 전기 자료를 공급하고, 거래처·부서/사원·프로젝트·현장·외주처별 초기이월(SA-OPN-07·08·13·17~19)이 각 보조부(P04)의 전기이월 잔액을 계정×차원 키로 적재한다(§25.2, §25.3). 온보딩 10단계 중 **⑨단계 "기준정보·개시잔액 적재"의 개시잔액 부분과 ⑩ 업무 개시 전 검증 게이트**를 담당한다(§22).

### 1.2 MVP 연관

- MVP 최우선(일반전표입력 WJV-01 + 계정과목 관리 WJV-08/09, §22 N13)의 직접 대상은 아니나, **P03 전표는 개시잔액 검증 게이트 통과를 선행 조건**으로 하므로(00_개요 §2) 본 페이즈의 검증 게이트(차대 균형, §22)가 MVP 업무 개시의 관문이다.
- 최초 작업년도에만 수기 입력하며, 차기부터는 SA-BAS-14 마감후이월(P01)이 본 페이즈 화면군의 데이터를 자동 적재한다(§25.2.1~§25.3.11 공통, §25.1.14).

### 3.2 개시잔액 검증 게이트 (§22, 부록 B §21, 부록 C §22)

1. 회사·사업장·회계기간 등록(P01) → 2. 계정체계 템플릿 매핑 → 3. 거래처/부서/프로젝트 대량 업로드(P01) → 4. **개시잔액(`OpeningBalance`) 업로드** → 5. 미결 채권채무/어음/자산 이관 → 6. **개시 시점 시산표 균형 검증** → 7. 검증 통과 시 `ACTIVE` 전환 (부록 C §22 EXT-14).

- **개시잔액 차대변 불균형 시 `ACTIVE` 전환을 차단**한다(균형 검증 게이트, §22·부록 B §21·부록 C §22).
- 미결 채권채무·어음·고정자산 이관은 `MigrationBatch`/`OpeningBalance` 단위로 검증·적재한다(§22).
- 대량 업로드는 **양식 검증 → 오류행 리포트 → 미리보기 → 일괄 반영**(`MigrationBatch`) 순서를 강제한다(부록 C §22).
- 계정 단위 균형 외에 **거래처별 초기이월 합계=계정 잔액 정합**(SA-OPN-07 좌우 차액 0, §25.2.7)·부서/사원·프로젝트·현장 차원 합계 정합(§25.3.5·§25.3.11)을 게이트 항목에 포함한다(8절 검증 표).

### 3.4 마감후이월(SA-BAS-14)과의 관계 (§25.1.14)

- 본 페이즈 전 화면은 "최초 작업년도에만 입력"이 원칙이며, **차기부터 SA-BAS-14가 마감기수 계정·차원별 종료월 잔액을 스냅샷하여 이월기수 `opening_balance`로 자동 적재**한다(직접 수정 가능). idempotent 재이월(기존 삭제 후 재적재) 지원(§25.1.14).
- 체크박스별 선택 이월: 재무제표(기존 삭제 후 덮어쓰기 → SA-OPN-01~06), 거래처/부서·사원/미완성공사/현장별·거래처별/외주처별/프로젝트별(종료월 잔액 → SA-OPN-07·08·13·17~19 전기이월 금액). 거래처별초기이월 "옵션" 제외 계정은 총 잔액으로 이월, **계정잔액 0원이면 거래처 잔액 미이월**(§25.1.14).
- 마감 상태 플래그(fiscal_period 단위 잠금)가 전표 수정·코드변환(SA-BAS-13)의 가드 조건 = **이월 잠금**. 마감(`CLOSED`/`LOCKED`) 기수의 전기이월 데이터 소급 수정은 마감해제 승인 워크플로우를 요구한다(§25.1.14, §15 "`LOCKED` 기간 마감해제 시도 차단", §12.2 회계기간 상태).

- **목적**: 계정과목별 **현장 초기이월** 및 **현장별 거래처 초기이월** 입력. 계정별원장(P04)의 현장 차원 및 현장×거래처 다차원 opening balance — 계정×현장(×거래처) 키 적재, **차액 검증을 서버 측 validation으로 enforce**.

**입력 필드**

| # | 필드 | 형식/값 | 검증·비고 (§25.3.11) |
|---|---|---|---|
| ① | 계정과목별 현장초기이월 | F2 | 전기분재무상태표(SA-OPN-01) 계정코드 선택→금액 반영, 우측 현장별 이월잔액 입력. **현장별 합계 vs 재무상태표 계정 차액을 [차액] 표시** |
| ② | 현장별 거래처초기이월 | F2 | 계정과목·현장 자동반영, 거래처 코드 F2로 우측 거래처별 이월잔액 입력. **현장별 합계 vs 현장 거래처별 합계 차액 표시** |
| ③ | 차액표시 | 여/부 | "여" 선택 시 차액 란 생성 — 전기분재무상태표 계정별 합계 vs 입력 현장별 계정 합계 비교 |

- **기능·검증**: 계정×현장×거래처 다차원 초기이월 입력·차액 표시. 전기분재무상태표 계정코드 기준, 현장별·거래처별 합계와 차액 정합성 검증(**서버 측 validation** — 게이트 항목, §25.3.11).
- **산출물**: 계정별원장 현장별 탭, 현장별 잔액명세서 전기이월(P04).
- **연계**: 선행 SA-OPN-01(준거)·SA-BAS-08(현장)·SA-BAS-06(거래처)·SA-BAS-14. 후행: 계정별원장 현장 차원 보조부.

## 11. 완료 기준(DoD) · 테스트 관점

1. **게이트**: 개시잔액 차대 불균형 시 `ACTIVE` 전환이 차단되고, 균형·정합(거래처/부서·사원/프로젝트/현장 합계=계정 잔액) 통과 시에만 전환된다(§22). 게이트 결과 API가 불균형 항목 목록을 반환한다.
2. SA-OPN-01: 대차차액 0 강제(종료 시 확인창+서버 제약), 법인/개인 자본 검증(375/376·379/380 규칙, 개인 자본금 합계), 기말재고→SA-OPN-02·03 반영, 이월이익잉여금↔SA-OPN-04 정합 검증이 동작한다(§25.2.1).
3. SA-OPN-02~04: 당기제품제조원가(02↔03)·당기순손익(02↔04) 교차 정합, 원가설정(관계 메타데이터 재사용), 처분계산서 라인추가·초기화가 동작한다(§25.2.2~4). SA-OPN-04는 개인사업자 테넌트에서 미노출.
4. SA-OPN-07: 전기분재무상태표 불러오기, 자금관리 계정(110·252·260) 관리내역 강제, 지급어음 선행 어음등록, 좌우 차액 0 검증 후 거래처원장·어음 보조부(P04)에서 전기이월 확인(§25.2.7).
5. SA-OPN-08·13·19: 사원→부서 자동반영, 계정×프로젝트×(부서/사원/거래처)·계정×현장(×거래처) 다차원 적재와 차액 서버 검증(§25.2.8, §25.3.5·11).
6. SA-OPN-09~12: 준거 금액 read-only, 월별/프로젝트별 합계 불일치 붉은색 경고, 계정과목복사·불러오기 동작(§25.3.1~4).
7. 대량 업로드: 양식 오류행 리포트→미리보기→일괄 반영(`MigrationBatch`), 멱등 재실행 시 중복 미발생(§22, 부록 C §22).
8. 기장 이관(2차 병행 가능): 전임 COA 매핑→GL/TB 수신→tie-out 전건 통과→기장 개시, 전임처 식별자 보존 분개 적재, `BookkeepingHandover` 감사기록(§22.1).
9. 2차: 중단사업 화면 환경설정 전제 게이팅(§25.3.6~7), 중도시산표 개시 분개 생성(미리보기·더존식 채번·멱등, §25.3.8), 미완성공사·외주처(잔액 자동계산)·현장별 초기이월(§25.3.9~11).
10. SA-BAS-14 순환: P01 마감후이월 실행 후 이월기수의 본 페이즈 화면에서 자동 적재분 조회·직접 수정, 재이월 시 기존 삭제 후 재적재(idempotent), 잔액 0원 스킵 확인(§25.1.14).
11. 비기능: RLS 테넌트 격리(교차 조회 0건), `DataChangeLog` 전후값, 표준 오류코드 응답, 마감 잠금 가드(00_개요 3절, §15).
12. 테스트 관점: 법인/개인 분기(SA-OPN-04 노출), 세목 사용/미사용(3/5자리), 차원 스위치 조합(프로젝트·현장 미사용 시 화면 비활성), 3개 연도(전기/전전기/전전전기) 입력, 게이트 불균형→보정→통과 시나리오, 기장 모드 3종별 입력 주체.
