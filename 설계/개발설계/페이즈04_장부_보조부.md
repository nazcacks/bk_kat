## 1. 페이즈 개요

### 1.1 목표

- P03 전표 도메인에 누적된 분개(`JournalLine`)를 **읽기 전용 뷰/집계 리포트로 투영**하는 주요장부(SA-LDG)와, 전표 라인의 관리항목(`journal_line_aux`)으로 입력되어 보조원장으로 집계되는 보조부(AUX)를 구축한다(§25.6 서두, §11.11 보조부 화면 정합, §12.1).
- `POSTED`(승인) 이상 전표만 집계하고, 6 관리차원(사업장 필수·코스트센터·부서·프로젝트·현장·사원)과 거래처를 GROUP BY 축으로 사용한다(§11.2, §25.6 서두).
- 채권/채무는 거래처 잔액 합계가 아니라 **Invoice(청구건) 단위 미결항목(Open Item)** 으로 관리하고 반제(Clearing)·부분입금·외화반제를 지원한다(§11.11 N4 — 2026-07-02 회의 결정 5).
- 은행 잔액 자동 대사·은행거래 분류·주간 자금마감·aging 스냅샷으로 P05 결산의 마감 체크리스트/계정별 명세 tie-out에 정합 데이터를 공급한다(§11.11, §11.9-(2), §11.14).

### 1.3 선행조건·MVP 연관

- P03 일반전표입력(WJV-01/SA-JNL-01)·매입매출전표입력(SA-JNL-02)이 MVP 최우선이며(§22, 회의 결정 1·2·10), 본 페이즈 장부는 그 산출 데이터의 1차 소비자다.
- 차원별 탭(부서/사원/현장/프로젝트)의 기초잔액은 P02의 [거래처별초기이월]·[부서사원별초기이월]·[현장별·거래처별초기이월]·[프로젝트초기이월]에서 로드한다(§25.6.5, §25.6.6, §25.7.5).
- 기장 모드(`TENANT_LED`/`OPERATOR_LED`/`HYBRID`)에 따라 조회·입력 권한이 분기된다(§21, 공통 표준 참조).

```

   │
   ├─▶ 주요장부 투영(읽기 전용): 전표출력→분개장→일/월계표→총계정원장→계정별원장
   │        └─ Drill-down: 금액 셀 → 전표 → 증빙 → 원거래(수집 원천) (§11.2 R19)

   │        └─ 미반제 Invoice 합계 = 거래처원장 잔액 상시 대사 (§11.11 N4)

   │        └─ 미매칭·미증빙 거래 → 미결항목(18.3) 승격 → [P05] 마감 게이트 (§11.11, §11.9)
   ├─▶ 부가세 축: 매입매출장(LDG-09) · AUX-06 부가세 보조원장(ntsStatus 대사) → [P06] 신고
   ├─▶ 승용차 축: 운행기록부(LDG-17) → 차량비용현황(LDG-18) → [P08] 업무용승용차 명세서
   └─▶ (2차) 원가/손익 안분(LDG-19·21, CostAllocationRule) → 안분전표(CLOSING_DR/CR)
            → 현황·공사원가·누적손익/원가(LDG-20·22~29) → [P05] 결산 연계
월말 배치: aging 스냅샷(AgingSnapshot) · 은행 자동 대사 · 주간 자금마감(WeeklyCashReport) (§11.11, §14)
```

### 4.1 AUX-01 거래처별 보조원장 (1차)

- **목적**: 매출처/매입처 원장. 거래성격·외화 정보·전표 연계를 라인 단위로 표시하고, Invoice 단위 미결항목(Open Item)과 잔액 정합을 유지한다(§11.11 보조부 화면 정합, N4).
- **진입경로**: 장부/보조부 > 거래처별 보조원장. 전표 화면 우측 패널·거래처원장(LDG-07)에서 상호 드릴다운.
- **조회조건**:

| 필드 | 유형/값 | 필수 | 비고(근거) |
|---|---|---|---|
| 기간 | 일자 범위(회계기간 내) | Y | §11.2 |
| 거래처 | F2/C 코드도움, 공란=전체 | N | §25.6.7 패턴 |
| 계정과목 | 채권/채무 계정(F2/C) | N | §11.11 |
| 조회 세분화 | 잔액만 / 연체건만 / 미반제건만 / 선수·선급 대체건만(조합 가능) | N | §11.11 M10 |
| 거래성격 | 신규채권 / 일부회수 / 청구반제 / 대손·조정 / 선수선급대체 | N | §11.11 M9 |
| 통화 | 전체/KRW/외화 통화별 | N | §11.11 M9 |
| 차원 | 사업장(필수)+코스트센터·부서·프로젝트·현장·사원 | 사업장 Y | §25.6 서두 |

- **표시(라인) 필드**: 전표 연계(`YYYY-MM-DD-NNN`)·일자·계정·적요·차/대변·잔액 + **외화금액·발생 시 적용환율**(라인 보존)·거래성격 플래그·Invoice 번호·`openAmount`·dueDate·증빙 링크(§11.11 M9·N4).
- **패널**: 거래처 현재 잔액·최장 연체일·연계 전표 요약(§11.11).
- **기능·동작**:

| 기능 | 동작 | 근거 |
|---|---|---|
| Invoice 관리 | 매출/매입 전표(또는 세금계산서·외부 인보이스)에서 `ArApInvoice`(invoiceNo, partner, currency, 외화/원화금액, 발생환율, dueDate, openAmount) 생성, 채권/채무 라인과 연결 | §11.11 N4 |
| 반제(Clearing) | 입금/지급 전표 입력 시 미반제 Invoice 목록 팝업/패널 제시 → 자동 매칭 제안(금액·인보이스 번호·적요) → 사용자 확정 | §11.11 N4 |
| 부분 입금 | Invoice `openAmount` 감액·잔액 추적. 초과 입금은 선수금 대체 제안(R5 선수금/선급금 자동 검토 연계) | §11.11 N4, §11.1.1-(5) |
| 외화 반제 | 반제 시점 환율 vs 발생환율 차이 → 환차손익 자동 계산, 은행(송금)수수료 별도 계정(지급수수료 등) 분리 분개. `ClearingEntry`(invoiceRef, journalRef, clearedAmount, fxGainLoss, fee) 보존 | §11.11 N4·M9, §11.1.1-(3) |
| 잔액 정합 대사 | 미반제 Invoice 합계 = 거래처원장(AUX-01) 잔액 상시 대사, 불일치 시 검토대상 | §11.11 N4 |

- **산출물**: 안분전표([일반전표입력] 추가). 현장코드 누락 전표는 [원가현황] '0000 누락코드' 조회(§25.8.1). **연계**: 후행 LDG-20·23·24, N12 비용 배부(§11.19-(3))와 단일 엔진.

### 4.29 SA-LDG-20 원가현황 / 4.30 SA-LDG-21 손익안분 / 4.31 SA-LDG-22 손익현황 (2차)

- **LDG-20 원가현황**: 원가안분·전표추가 후 차원별 원가 조회. 조회기간·차원(F2/C)·경비구분(1.50000번대 제조/2.60000번대 공사/3.70000번대 분양)·원가설정(매출원가코드·원가경비코드)·상세/일괄조회설정. 안분 산출 전표+직접 차원 전표를 경비구분별 집계, 원장조회·보고서(원가보고서 조회·인쇄)(§25.8.2).
- **LDG-21 손익안분**: [손익계산서] 금액 합계 반영, 차원 미입력 판매비와 관리비(간접비) 안분. 조회조건 = 기간·차액·계정과목([손익계산서])·안분/분개·합계·기초 상품 재고액·당기 상품 매입액. **안분기준·산식·원단위 잔차·역분개+안분전표 생성은 LDG-19와 동일 엔진**(`CostAllocationRule` 손익 안분, 단일 엔진 공유). 산출: 안분전표, 누락 전표 '0000 누락코드'(§25.8.3).
- **LDG-22 손익현황**: 손익안분 후 차원별 P&L 집계. 조회기간·차원(F2/C)·계정과목코드보기. 손익안분 산출 전표+직접 차원 전표 집계, 원장조회(§25.8.4).

### 4.32 SA-LDG-23 원가손익 (2차)

- **목적**: 차원별 원가손익(매출·매입계정·원재료·노무비·경비(원가)·판관비) 월별 조회(§25.8.5).
- **조회조건**: 조회기간·차원(F2/C)·구분(원가경비구분)·과목/세목·재고자산설정('선택' V 체크 계정은 'II.매입계정' 표시)·원가설정(매출원가-원가경비)·원장조회·계정과목코드보기(§25.8.5).
- **기능**: 원가안분(LDG-19)+손익안분(LDG-21) 결과 결합, 재고자산·원가경비 매핑으로 매입계정/매출원가 구성, 월별 원가손익현황(§25.8.5).

### 4.33 SA-LDG-24 현장별 공사원가집계표 (2차)

- **목적**: 현장별 공사기간·공사금액·기성내역·공사비내역. 공사원가집계표/작업진행률 탭(§25.8.6).
- **조회조건**: 기간·원가구분(0.도급공사원가/1.분양공사원가)·현장(F2/C)·조회([확인]=저장 데이터/[취소]=재계산)·Code/현장명/공사기간/공사금액([현장등록] 반영)(§25.8.6).
- **산식**: 기성내역 = 전기 기성액(전기 '총기성금액 이월'/'전기기성금액 조회' 편집) + 당기 기성액('매출계정등록' 계정, 기본 '407 공사수입금') = 총 기성금액. 공사비내역 = 전기 공사비([미완성공사 초기이월]·'전기공사비 반영설정') + 당기 공사비(재료비+노무비+경비) = 총공사누적액. 당기 공사비 = 현장별 전표 입력금액 + [원가안분] 안분 전표. 재료비=원(부)재료비, 노무비=[계정과목] 구분 '노무비', 경비=구분 '경비/기타'(§25.8.6).
- **작업진행률**: 진행률 = 총공사누적비 ÷ 총공사예정비(직접입력), 기성고계산액 = 공사계약금액 × 진행률, 조정액 = 기성고계산액 - (전기말 + 당기 회사수입계상액)(§25.8.6).
- **기능(더보기)**: 원가설정(변경 시 [미완성공사 초기이월] 삭제)·전기공사비 반영설정(총금액/상세금액)·공사금액기준보기·총기성금액 이월·현장별공사비 조회·당기공사원가수정('세부내역 반영')(§25.8.6).

### 4.34 SA-LDG-25 공사비용현황 / 4.35 SA-LDG-26 현장별 잔액명세서 (2차)

- **LDG-25**: 현장별 원가비용 현황. 기간·현장(F2/C)·과목/세목·원가설정(매출원가·원가경비코드 연결)·계정과목코드보기·원장조회. 원가경비 계정 범위를 현장 축 집계(§25.8.7).
- **LDG-26**: 현장별 모든 계정과목 잔액. 현장별/계정별 축 전환(현장별=계정의 현장별 잔액+공사내역, 계정별=현장의 계정별 잔액+공사내역)·기간·현장·계정과목(각 F2/C, 공란=전체). 기초잔액=[현장별·거래처별초기이월] "계정과목별 현장초기이월" 탭. 현장 × 계정 잔액(기초+증감)·공사내역 동반 표시(§25.8.8).

### 4.36 SA-LDG-27 차원 거래처원장 (2차)

- **목적**: 현장·부서/사원·프로젝트별 × 계정과목 × 거래처 잔액·거래내용. 잔액/내용/총괄잔액/총괄내용 4탭(LDG-07 구조의 차원 축 확장, §25.8.9 충돌 조정).
- **조회조건**: 기간·구분/구분코드(0.현장별/1.부서별/2.사원별/3.프로젝트별 + 코드, 공란=전체)·**계정과목(필수, 두 글자 이상/코드 직접입력)**·거래처(F2/C)·과목/세목. 전기(월)이월 = [현장별·거래처별]/[부서사원별]/[프로젝트] 초기이월(§25.8.9).
- **기능**: 더보기 — 잔액표시방법(잔액없는 거래처 표시·건별잔액·음수 빨간색)·미등록거래처표기·잔액집계(잔액 있는 거래처만)·전표수정(내용/총괄내용 더블클릭)(§25.8.9).

### 4.37 SA-LDG-28 프로젝트 누적손익현황 / 4.38 SA-LDG-29 프로젝트 누적원가현황 (2차)

- **LDG-28**: 프로젝트별 월별 손익 + 누적. "전기 누적액"=[전기프로젝트손익현황], "당기 누적액"=전기 누적액+당기 발생액. 조회: 구분(0.전체/1.진행/2.보류)·프로젝트(F2/C)·조회기간·과목/세목·증감률계산설정(소수점·계산방법)·원장조회·전체프로젝트 인쇄. 원천 = 프로젝트 차원 P&L 전표 + 손익안분 산출분(LDG-21)(§25.8.10).
- **LDG-29**: 프로젝트별 월별 원가 + 누적. "전기 누적액"=[전기프로젝트원가현황] 자동 반영. 조회: 조회기간·구분(진행상태)·프로젝트·원가경비(0.50000 제조/1.60000 공사/2.70000 분양)·과목/세목·증감률계산설정·원가설정·원장조회. 원천 = 프로젝트 차원 원가 전표 + 원가안분 산출분(LDG-19)(§25.8.11).
