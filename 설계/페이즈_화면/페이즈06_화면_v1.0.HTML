§3End-to-End 신고 흐름

P03 전표·수집자료→ 부속서류·합계표 작성→ 신고서 생성(예정/확정)→ 마감 · MRI 60여 항목→ 이용회사 Ack·승인(Step-up)→ 전자신고 .101/.102→ 납부서 2D · LOCKED

기존 문서형 화면을 P03 전표입력 화면과 동일한 밀도로 재구성했다. 모든 업무 화면에 조회조건(qbar)·액션바·필요 시 탭·대표 데이터 그리드·우측 상태/규칙 패널을 배치하고, 화면 설명은 목업 하단 설계카드로 분리했다.

핵심 설계 포인트

단일 원천P03 매입매출전표 과세유형 11~64가 신고서식 라우팅의 단일 진실원. 매출 11~25 / 매입 51~64 코드별 자동 집계.
마감 상태머신작성중 → 마감 → 첨부서식 조회 → 전자신고여부(0/x) → MRI 검사. 차단 오류는 마감 저지, 유의사항은 강제마감 허용.
과세단위사업장별 / 총괄납부 / 사업자단위 — 6차원 중 사업장 차원이 과세단위 분기 키.
이용회사 승인filingApprovalRequired=Y이면 전자신고 전 Ack·Step-up 인증 필요.
전자신고제출자등록 → 제작(.101 일반/.102 간이) → 검증 → 홈택스 제출(약15단계). 상태머신 6종.
